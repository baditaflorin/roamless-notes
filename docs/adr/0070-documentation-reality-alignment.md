# 0070 Documentation Reality Alignment Process

## Status

Accepted

## Context

README and app claims must match shipped behavior.

## Decision

Every README feature claim must correspond to a test or an audited manual verification. Claims that are aspirational are moved to limitations or deleted.

## Consequences

The README becomes shorter, more useful, and less promotional.

## Alternatives Considered

Leaving future-tense claims was rejected because it confuses strangers.
